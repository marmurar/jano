# Spec: Walk-Forward Runner

## Purpose

`WalkForwardRunner` executes a model over temporal folds generated by a Jano workflow.
It answers operational questions such as:

- What happens if the model retrains on every fold?
- What happens if the model is trained once and reused?
- What happens if retraining happens every `K` folds?
- What happens if retraining depends on previously observed fold metrics?

The runner extends simulation capabilities without changing the splitter core.

## Public Inputs

The runner accepts:

- an estimator with `fit()` and `predict()`
- `target_col` or a separate `y`
- optional `feature_cols`
- retraining configuration
- metric names or custom metric functions

Supported workflow inputs include:

- `WalkForwardPolicy`
- `TemporalSimulation`
- `TemporalBacktestSplitter`
- compatible objects exposing `simulation` or `as_splitter()`

## Retraining Policies

Supported retraining modes:

- `retrain=True` or `retrain="always"`
- `retrain=False` or `retrain="never"`
- `retrain="periodic"` with `retrain_interval=K`
- explicit `RetrainPolicy` objects such as `DriftBasedRetrain`

Retraining decisions may only use information observed up to the previous fold.

## Outputs

`WalkForwardRunResult` exposes:

- `to_frame()` for fold-level execution records
- `fold_summary()` for geometry and retraining metadata
- `metric_trajectory()` for long-format metrics
- `retrain_events()` for folds where the estimator was refit
- `predictions_frame()` for row-level test predictions
- `summary()` for compact aggregate stats
- `report_data()` and `to_dict()` for agent-friendly structured output

## Non-Goals

The runner does not:

- redefine fold geometry
- mutate splitter behavior
- choose a model
- infer a target type by default
- own final visualization or dashboard generation

## Acceptance Criteria

- Running the same workflow with different retraining policies keeps the same fold geometry.
- The first fold always has a fitted model before prediction.
- `NeverRetrain` trains once and reuses the fitted estimator.
- `PeriodicRetrain` retrains at the configured cadence.
- `DriftBasedRetrain` uses previous fold history only.
- Result outputs are pandas-friendly and serializable for external visualization.
